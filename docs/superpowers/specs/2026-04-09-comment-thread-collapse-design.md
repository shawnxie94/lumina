# Comment 线程嵌套折叠设计

## 目标

为 `CommentSection` 增加分层折叠能力：保留顶层评论的“查看回复/收起回复”入口，同时让任意一条拥有子回复的回复也能单独折叠/展开其直接子回复，改善深层评论线程的可读性。

## 交互约束

- 顶层评论默认折叠整条回复线程。
- 顶层展开后，二级及更深层回复默认保持折叠。
- 任意一条回复若存在子回复，显示与顶层一致风格的折叠开关。
- 顶层开关继续展示整条线程的总回复数。
- 嵌套回复开关展示该回复的直接子回复数。
- 通过 hash、原评论链接或其他定位行为跳到深层评论时，自动展开沿途祖先回复，确保目标评论可见并能滚动高亮。

## 实现要点

1. 将 `expandedReplies` 的语义扩展为“任意评论 id 对应的子回复展开状态”。
2. 抽出祖先链解析逻辑，用于定位时一次性展开从顶层到目标评论父级的所有节点。
3. 在递归渲染回复线程时，为存在子回复的回复节点渲染自己的折叠开关，仅在展开时渲染直接子回复列表。
4. 保持现有提交、编辑、删除与回复前缀逻辑不变。

## 验证

- 回归测试覆盖深层回复的祖先展开路径。
- 回归测试覆盖顶层总回复数与嵌套层子回复开关文案。
- 前端测试与生产构建通过。
